## Strategy

Simple manual setup. No Ansible, no systemd yet. We'll:
1. SSH as root to install system packages
2. Create a `openrai` user for the producer
3. Install `uv` system-wide
4. Clone the repo, set up venv with `uv`
5. Generate keys, save to `~openrai/.env`
6. Test the pipeline manually

No Nano node yet means we can't test the full `snapshot` step yet (needs a live `data.ldb`). But we can test key generation, `publish --dry-run`, and the overall setup.

**Note on libtorrent**: The `publish` step requires `libtorrent` for DHT publishing. This is a C++ library that can't be pip-installed easily on bare metal. Options:
- **Option A (recommended for testing)**: Use `publish --dry-run` which doesn't need libtorrent
- **Option B**: Install libtorrent from source (same build as Dockerfile, ~5 min)
- **Option C**: Build the Docker image and use the container for the publish step

For first-time testing, we'll use `--dry-run` and defer the full DHT publish to when we're ready to use Docker for that step.

### Step 5: Generate Ed25519 key pair (as openrai)

```bash
cd /opt/nano-snapshot-swarm
source .venv/bin/activate

python -c "
from nacl.signing import SigningKey
sk = SigningKey.generate()
print(f'DHT_PRIVATE_KEY={sk.encode().hex()}')
print(f'AUTHORITY_PUBKEY={sk.verify_key.encode().hex()}')
"
```

Save output to `~openrai/.env`:

```bash
cat > ~openrai/.env << 'EOF'
export DHT_PRIVATE_KEY=<paste_private_key>
export AUTHORITY_PUBKEY=<paste_public_key>
export NANO_LEDGER_PATH=/var/nano/data/data.ldb
export OUTPUT_DIR=/opt/nano-snapshots
EOF
chmod 600 ~openrai/.env
```

### Step 5: Build libtorrent from source (as root, installs to /usr/local)

```bash
cd /tmp
git clone --depth 1 --branch v2.0.11 --recurse-submodules https://github.com/arvidn/libtorrent.git
cd libtorrent
mkdir build && cd build
cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_INSTALL_PREFIX=/usr/local \
  -DBUILD_SHARED_LIBS=ON \
  -DBUILD_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -Dpython-bindings=ON \
  -Dpython-install-system-dir=ON
make -j$(nproc)
make install
ldconfig
python3 -c "import libtorrent; print(libtorrent.version)"  # verify
rm -rf /tmp/libtorrent
```

Note: Ubuntu 24.04 has Python 3.12 and `libboost-python-dev` should match. If cmake picks the wrong Python, we may need `-DPython3_EXECUTABLE=/usr/bin/python3`. We'll verify after the build.
