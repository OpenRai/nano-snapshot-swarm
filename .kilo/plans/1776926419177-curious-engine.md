# Plan: Deploy the Public Dashboard

## Current State

All code exists and is functional:

- **`status-api/`** — FastAPI app with `/`, `/api/status`, `/api/status-fragment`, `/api/torrent`, `/api/push`, `/health` endpoints. Dashboard HTML is built into `app/templates/index.html` using HTMX.
- **`dashboard/index.html`** — Standalone static page that hardcodes `https://nano-snapshot-hub.fly.dev/api/status-fragment`. This is redundant with the status-api's built-in dashboard but could serve as a static fallback (e.g., Cloudflare Pages or GitHub Pages).
- **`producer/push_status.py`** — CLI that signs + pushes snapshot data to the status API.
- **`systemd/nano-status-push.{service,timer}`** — Hourly timer to push status from the producer server.
- **`scripts/push-snapshot-status.sh`** — Shell wrapper for push_status.py.
- **`status-api/fly.toml`** — Fly.io config for app `nano-snapshot-hub`.
- **`status-api/deploy/fly.io/README.md`** — Complete deployment runbook.

**What's NOT done:**
- The Fly.io app `nano-snapshot-hub` does not exist yet (DNS doesn't resolve).
- No CI/CD workflow for deploying the status-api to Fly.io.
- The producer on `bandwidth-martyr` doesn't have `STATUS_API_URL` configured, so `push-snapshot-status.sh` exits silently.
- The `dashboard/index.html` hardcodes the Fly.dev URL and isn't served anywhere.

## Steps to Deploy

### Step 1: Create and deploy the Fly.io app

Follow the runbook at `status-api/deploy/fly.io/README.md`:

```bash
cd status-api
fly auth login
fly apps create nano-snapshot-hub
fly volumes create status_data --size 1 --region sjc --app nano-snapshot-hub
fly deploy
```

### Step 2: Verify the deployment

```bash
fly status --app nano-snapshot-hub
curl https://nano-snapshot-hub.fly.dev/health
# Should return {"status":"ok","sequence":0,"updated_at":""}
curl https://nano-snapshot-hub.fly.dev/
# Should return the dashboard HTML (HTMX will poll /api/status-fragment every 60s)
```

Initially `/api/status` and `/api/torrent` return 404 until the producer pushes data.

### Step 3: Configure the producer to push status

On the `bandwidth-martyr` server, add `STATUS_API_URL` to `~/.env`:

```bash
echo 'STATUS_API_URL=https://nano-snapshot-hub.fly.dev' >> ~/.env
```

Then enable and start the timer:

```bash
systemctl --user enable nano-status-push.timer
systemctl --user start nano-status-push.timer
```

### Step 4: Test the first push manually

```bash
systemctl --user start nano-status-push.service
journalctl --user -u nano-status-push.service -n 20 --no-pager
```

Then verify the dashboard shows data:

```bash
curl https://nano-snapshot-hub.fly.dev/api/status | python3 -m json.tool
```

### Step 5: Add CI/CD workflow

Create `.github/workflows/deploy-status-api.yml` per the runbook (section 6.3). This auto-deploys on pushes to `main` that touch `status-api/`.

Steps:
1. Generate a Fly API token: `fly tokens create deploy -x 999999h`
2. Add `FLY_API_TOKEN` as a GitHub secret in repo Settings → Secrets → Actions
3. Create the workflow file:

```yaml
name: Deploy Status API

on:
  push:
    branches: [main]
    paths:
      - "status-api/**"

jobs:
  deploy:
    name: Deploy to Fly.io
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: superfly/flyctl-actions/setup-flyctl@master
      - name: Deploy
        run: fly deploy --config status-api/fly.toml
        env:
          FLY_API_TOKEN: ${{ secrets.FLY_API_TOKEN }}
```

### Step 6: Keep `dashboard/index.html` as static fallback

The `dashboard/` directory contains a standalone HTML page that hardcodes `https://nano-snapshot-hub.fly.dev/api/status-fragment`. It duplicates the status-api's built-in dashboard but can serve as a CDN-resilient fallback. Deploy it to Cloudflare Pages or GitHub Pages so the dashboard stays accessible even if Fly.io is down.

Action item: Add a Cloudflare Pages or GitHub Pages deployment for `dashboard/`.

### Step 7: Set up Cloudflare DNS (optional but recommended)

Per the runbook (section 5), point a custom domain like `snapshots.openrai.org` to the Fly app for a polished URL and Cloudflare caching:

1. `fly certs create snapshots.openrai.org --app nano-snapshot-hub`
2. Add CNAME in Cloudflare: `snapshots → nano-snapshot-hub.fly.dev`
3. Set Cloudflare SSL to Full (strict)
4. Add cache rules for `/api/torrent` (1h TTL) and `/api/status*` (5min TTL)

This reduces Fly billable bandwidth to near-zero.
